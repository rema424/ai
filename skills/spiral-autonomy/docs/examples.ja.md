# Spiral Autonomy プロンプト例

これらは起動プロンプトの雛形です。タスク固有の目的や制約はプロンプトに書き、再利用するプロセスは Skill に任せます。

## 最小

```text
$spiral-autonomy を使ってこのタスクを進めてください。

目的:
<実現したいこと>
```

## 長時間の自律実行

```text
$spiral-autonomy を使って、長時間の自律実行として進めてください。

目的:
<最終的に到達したい状態>

背景:
<repo、プロダクト、事業、プロジェクトの前提>

制約:
<期限、非目標、リスク、使えるツール、環境>

期待する成果物:
<文書、コード、計画、意思決定パッケージ、レビュー結果など>
```

## 調査

```text
$spiral-autonomy を profile=research で使ってください。

調査したい問い:
<調査テーマ>

範囲:
<対象範囲と対象外>

証拠要件:
<一次情報、鮮度、確度、反証の扱い>

出力:
確認済み事実、推測、未確認事項、確度、次の検証手段。
```

## 事業計画

```text
$spiral-autonomy を profile=business-plan で使ってください。

目的:
次の事業アイデアを評価し、必要なら修正してください。
<アイデア>

重点:
顧客、課題の切迫度、提供価値、市場仮説、収益モデル、GTM、検証実験。

出力:
更新済み事業計画、最も弱い仮説、検証計画、人間が判断すべき論点。
```

## プロダクト企画

```text
$spiral-autonomy を profile=product-planning で使ってください。

目的:
次のプロダクト構想を、実行可能なスコープに落としてください。
<構想>

重点:
ユーザー、JTBD、成功基準、MVP スコープ、非目標、リリース順序、学習マイルストーン。

出力:
プロダクト計画、スコープ判断、棄却した選択肢、次の検証サイクル。
```

## 要件定義

```text
$spiral-autonomy を profile=requirements で使ってください。

目的:
次の対象の要件を定義してください。
<システム、機能、業務フロー、プロジェクト>

既知の前提:
<事実、制約、ステークホルダー>

出力:
ステークホルダー、ユースケース、機能要件、非機能要件、制約、受け入れ基準、エッジケース、未確認事項。
```

## 設計

```text
$spiral-autonomy を profile=design で使ってください。

目的:
次の対象を設計してください。
<システム、プロセス、アーキテクチャ、業務フロー>

制約:
<技術、運用、事業、セキュリティ、コスト、期限>

出力:
代替案、採用案、棄却理由、インターフェース、境界、失敗モード、移行または展開計画、テスト戦略。
```

## 調査から設計までの Multi-Phase

```text
$spiral-autonomy を使って、調査から設計までの multi-phase spiral として進めてください。

目的:
<広い到達目標>

現時点の前提:
<今わかっていること>

期待する進め方:
調査、企画、要件定義、設計、必要なら spike 検証、再修正、handoff。

出力:
更新済みの調査 baseline、企画、要件、設計、spike 結果、学びによって変わった判断、checkpoint、残リスク、次の実行計画。
```

## Spike 検証

```text
$spiral-autonomy を使って、spike-driven validation として進めてください。

仮説:
<正しいかもしれない前提>

リスク:
<この不確実性が重要な理由>

許可する spike 形式:
<コード試作、ベンチマーク、データ分析、UX mock、一次情報確認、市場証拠確認など>

出力:
仮説、検証方法、証拠、結果、上流への修正、推奨判断。
```

## 実装

```text
$spiral-autonomy を profile=implementation で使ってください。

目的:
次を実装してください。
<変更内容>

制約:
<スコープ、対象ファイル、互換性、テスト、展開条件>

出力:
変更内容、検証結果、checkpoint、残リスク、実装中に見つかった要件・設計への修正。
```

## レビュー

```text
$spiral-autonomy を profile=review で使ってください。

レビュー対象:
<PR、ファイル、設計文書、計画、成果物>

レビュー観点:
正しさ、回帰、テスト不足、セキュリティ、運用リスク、証拠品質。

出力:
重大度順の findings first。根拠と修正案を含めてください。
```
